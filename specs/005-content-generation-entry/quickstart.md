# Quickstart & Validation: Content Generation Entry Point

**Feature**: 005-content-generation-entry | **Date**: 2026-07-06

This guide validates the feature end-to-end. It references the [navigation contract](./contracts/navigation.md)
(C-1…C-3, VC-1…VC-4) rather than restating it.

## Prerequisites

- Dependencies installed: `npm install`
- Feature implemented per `tasks.md` (generated by `/speckit-tasks`).

## Run the app

```bash
npm run dev
```

Then open `http://localhost:3000/`.

## Manual validation scenarios

### Scenario A — Control is present and on-brand (VC-1, FR-001/FR-002)

1. Load the dashboard home page (`/`).
2. **Expected**: A "Content Generation" control appears in the header toolbar area alongside the
   Previous/Today/Next range navigation, the Week/Month toggle, and the brand filters. It is
   styled like the other controls (filled/pill treatment, shared theme) — not a bare button.

### Scenario B — Client-side navigation to the page (C-1, FR-003)

1. From `/`, click the "Content Generation" control.
2. **Expected**: The browser moves to `/content-generation` without a full page reload (no white
   flash / full document reload; App Router transition).

### Scenario C — Placeholder page renders on-brand (VC-2, VC-3, FR-004/FR-005)

1. Observe the `/content-generation` page.
2. **Expected**: Non-white background (shared `slate-200` body theme), consistent typography, a
   "Content Generation" title, and a short "coming soon"-style message. Layout matches the item
   detail page's back-navigation header pattern — not a blank/unstyled page.

### Scenario D — Back to dashboard (C-2, FR-006)

1. On `/content-generation`, click "← Back to dashboard".
2. **Expected**: Returns to the dashboard home page (`/`).

### Scenario E — Direct visit / refresh (C-3, edge case)

1. Navigate directly to `http://localhost:3000/content-generation` (or refresh while on it).
2. **Expected**: The placeholder page still renders fully with theme, title, message, and a
   working back link.

### Scenario F — No functional logic present (VC-4, FR-008)

1. Inspect the `/content-generation` page.
2. **Expected**: No forms, text inputs, or generation/action buttons beyond the back-navigation
   link. Nothing triggers content creation, file writes, or AI calls.

## Automated checks (optional, matches existing harness)

- **Component/unit (Vitest)**: assert the dashboard header renders a link with text
  "Content Generation" whose `href` is `/content-generation`; assert the new page renders the
  title, a coming-soon message, and a back link with `href="/"`.
- **E2E (Playwright)**: drive Scenarios B, D, and E — click through from `/`, assert URL
  `/content-generation` and visible title, click back, assert URL `/`; then load
  `/content-generation` directly and assert it renders.

## Success = all scenarios pass

Maps to spec Success Criteria SC-001…SC-006. When every scenario above passes, the navigation
shell is complete and ready for the future content-generation feature to build on.
